# 📋 01 — Requirements & User Flows
---

## 📝 1. Обзор

> Приложение, для решения проблем с нехваткой времени и организованностью.
> Интерактивный менеджер задач для группы с простым интерфейсом позволит удобно
> создавать задания, видить их прогресс, статистику, требования и вклад каждого
> участника. Нацелено на работу с музыкой.

## 👥 2. Роли пользователей

| Роль | Описание | Права |
|---|---|---|
| Member | Обычный участник команды, зарегестрированный на сайте | Создаёт/редактирует свои задачи, видит задачи всех | 

>💡 На данный момент разделения на роли нет, у всех участников одинаковые права.

## ⚙️ 3. Функциональные требования

Что должна уметь выполнять каждая подчасть проекта. 

### ✅ 3.1 Задачи (Tasks)

- Пользователь может создать задачу с полями: 
  - 🏷️ **название**
  - 📝 **описание** (необязательно)
  - 📅 **конечный срок выполнения (DueDate)**
  - 📅 **дату создания (CreatedAt)**
  - 👤👥 **исполнитель/и**
  - 🚦 **статус** (подробнее ниже)
- 📂 Задача может содержать **подзадачи** 
- 🗓️ Задача отображается в календаре на отметке текущего времени и плавно перемещается относительно его
- ⏰ Просроченные задачи (дошедшие до DueDate срока, но не выполнениы) показываются отдельно от сегодняшних

### 📂 3.2 Подзадачи (Subtasks)

- 🔗 Подзадача принадлежит **одной задаче**
- Например: 
  - 🎵 Задача - сделать трек
  - 🎼 Подзадачи 
    - 🥁 написать бит
    - ✍️ придумать текст
    - 🎤 записаться
    - 🎚️ свести)
- Пользователь может создать подзадачу с полями: 
  - 🏷️ название
  - 🚦 **статус**
  - 👤👥 **исполнитель/и** ()

### 3.3 Статусы

- ⏳ Для выполнения
- 🔄 В процессе
- ✅ Готово
- ❌ Просрочено (Применяется автоматически)


### 📦 3.4 Выполненные задачи
- ✅ После завершения задача переносится в группу выполненных.
- 👤 Пользователь, который был указан выполняющим, получает "+" в статистику выполненных задач, когда задача становится выполненной
- 👥 Пользователь, который был указан выполняющим, получает "+" в статистику выполненных подзадач, когда задача становится выполненной

### 3.5 Невыполненные задачи
- ❌ Просроченные задачи переносятся в отдельную группу.
- 👥 Не успевший выполнить задачу/подзадачу исполнитель получает "-" в статистику выполненных подзадач
- 🔄 У пользователя есть возможность пересоздать невыполненную задачу. Старые данные предыдущей задачи сохраняются (кроме сбрасывающегося dueDate), но есть возможность моментально поменять определенные.

### 🔐 3.6 Аутентификация

- Приложение содержит простую аутентификация и всю работу с задачами могут осуществлять только зарегистрированные пользователи

## 🔄 4. User Flows

### 🔐 Flow 1 - Регистрация/Вход

1. 🛂 Пользователь регистрируется/входит в приложение по логину и паролю

### ✨ Flow 2 — Создание задачи

1. 📄 Открыть страницу
2. ➕ Нажать «Создать задание»
3. ✍️ Заполнить поля
4. 💾 Сохранить
5. 📋 Увидеть задачу в списке

### 🔨 Flow 3 — Выполнение задачи

1. ✅ Отметить подзадачу выполненной
2. 🗓️ С течением времени положение задачи меняется

### 🏁 Flow 4 — Завершение задачи

1. 📦 Выполненная задача переносится в архив выполненных.
2. ❌ Невыполненная — в список просроченных.

## 📏 5. Бизнес-правила и ограничения

Важные "если-то" правила, которые не очевидны из списка фич.

**Примеры вопросов, на которые стоит ответить здесь:**
- ✏️ Может ли пользователь редактировать чужую задачу?
  - Пользователь может редактировать только те задачи, которые созданны им же. 
- 🗑️ Можно ли удалить задачу, если у неё есть подзадачи?
  - Да, в таком случае подзадачи также будут удалены. 
- 👥 Что происходит с задачей, если удалили исполнителя (пользователя)?
  - В задаче можно удалять исполнителей до тех пор, пока не останется один. Задаче не может не иметь исполнителей.
- 📅 Может ли задача быть без конечной даты?
  - На данном этапе нет. При необходимости, такие задачи будут добавленны в более поздних версиях.
